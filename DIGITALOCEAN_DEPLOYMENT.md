# DigitalOcean App Platform Deployment Guide

## 🚀 Quick Start (Recommended)

### Prerequisites
- DigitalOcean account with App Platform access
- GitHub account connected to DigitalOcean
- OpenAI API key

### Step 1: Use the Quick Configuration
Use **`app.yaml.simple`** for a standard Backend + Dashboard deployment (recommended for most users).

### Step 2: Deploy via DigitalOcean Console

1. **Log in to DigitalOcean** → Apps → Create App
2. **Select GitHub** and choose:
   - Repository: `kamaraj/kyrochatbot`
   - Branch: `tcsaasbot`
   - Source directory: (leave empty - uses root)

3. **DigitalOcean will auto-detect** the `app.yaml` file

4. **Configure Environment Variables:**
   - `OPENAI_API_KEY` - Your OpenAI API key
   - `APP_DOMAIN` - Your app domain (e.g., tangentcloud.app)
   - `DATABASE_URL` - Auto-generated by DigitalOcean
   - `REDIS_URL` - Auto-generated by DigitalOcean

5. **Click Deploy!**

## 🔧 Manual Deployment (If Auto-Detect Fails)

### Option A: Use DigitalOcean CLI

```bash
# Install doctl
brew install doctl

# Login
doctl auth init

# Deploy from app.yaml.simple
doctl apps create --spec app.yaml.simple
```

### Option B: Manual Configuration in Console

1. **Create App** → Select GitHub repo
2. **Add Services** manually:
   - Backend: `source_dir: backend`
   - Dashboard: `source_dir: dashboard`
3. **Add Databases:**
   - PostgreSQL 15
   - Redis 7
4. **Set Environment Variables**
5. **Deploy**


## 🚨 Troubleshooting

### Issue: "No components detected"
**Solution:** Make sure `app.yaml` or `app.yaml.simple` is in the root directory (✅ it is now)

### Issue: Build fails - "requirements.txt not found"
**Solution:** Verify `backend/requirements.txt` exists
```bash
ls -la backend/requirements.txt
```

### Issue: Dashboard build fails - "package.json not found"
**Solution:** Verify `dashboard/package.json` exists
```bash
ls -la dashboard/package.json
```

### Issue: Database connection timeout
**Solution:** 
1. Check PostgreSQL is running: DigitalOcean Console → Databases → Status
2. Verify `DATABASE_URL` environment variable is correct
3. Restart the app

### Issue: API calls fail with 502 Bad Gateway
**Solution:**
1. Check backend health: `https://your-app.ondigitalocean.app/api/health`
2. View logs: DigitalOcean Console → Apps → Logs
3. Verify `OPENAI_API_KEY` is set correctly

### Issue: High memory usage
**Solution:**
1. Restart the app from console
2. Scale up machine size if needed
3. Check for memory leaks in logs

## 📈 Post-Deployment

### 1. Monitor Your App
- **DigitalOcean Console** → Apps → Your App → Insights
- **Health Checks:** Backend `/health`, Dashboard `/`
- **Resource Usage:** CPU, Memory, Network

### 2. Enable SSL/HTTPS
- Automatic with DigitalOcean App Platform
- HTTPS enabled by default at deployment

### 3. Configure Custom Domain
1. DigitalOcean Console → Apps → Domains
2. Add your domain
3. Update DNS records at your registrar

### 4. Set Up Backups
- **PostgreSQL:** Automatic daily backups (7 days)
- **Verify:** DigitalOcean Console → Databases → Backups

### 5. Configure Alerts
- Email notifications for deployment failures
- Low disk space warnings
- High CPU/Memory thresholds
